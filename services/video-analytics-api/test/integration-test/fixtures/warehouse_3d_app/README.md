# warehouse_3d_app fixtures

Fixtures for integration tests covering **warehouse_3d_app** with data from the Elasticsearch dump already present in the repo (time range **2026-02-23T10:09:22.753Z** to **2026-02-23T10:13:29.453Z**, place `room=SURF Booth/region=region-1`, sensor `bev-sensor-1`).

- **mdx-behavior**: GET `/behavior` — sensorId with timestamps, objectId, objectType, and maxResultSize combinations.
- **mdx-frames**: GET `/frames/enhanced`, `/frames/bev`, `/frames/alerts`, and `/frames/proximity-detection`.
- **mdx-events**: GET `/events/roi`.
- **metrics**: GET `/metrics/occupancy/fov`, `/metrics/occupancy/fov/histogram`, `/metrics/occupancy/roi`, and `/metrics/occupancy/roi/histogram`.

The 3D dump lives in the shared top-level Elasticsearch fixture files in this repo, not a separate `warehouse-3d/` subdirectory. The generator intentionally uses the `bev-sensor-1` records from those shared dumps.

Generated by (from repo root): `node test/integration-test/scripts/run_warehouse_3d_app_tests.js <BASE_URL> [WEB_APIS_ROOT] [FIXTURES_DIR]`  
Or from `test/integration-test`: `node scripts/run_warehouse_3d_app_tests.js <BASE_URL> [WEB_APIS_ROOT] [FIXTURES_DIR]`

Each subdirectory contains:
- `input.json` — query parameters used for the request (validated with OpenAPI/AJV).
- `output.json` — response body (validated with OpenAPI spec).

The 3D-specific scope in this directory is intentionally limited to the dump-backed endpoints above.

The remaining 3D-required config endpoints stay covered by the shared config integration flow in this repo instead of this dump generator:
- GET `/config/usd-assets`
- POST `/config/upload-file/usd-assets`
- POST `/config/upload-file/calibration`
- GET `/config/calibration/image`
- POST `/config/calibration/images`
